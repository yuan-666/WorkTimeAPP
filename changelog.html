Changelog

产品更新轨迹

这里按版本记录真正影响使用体验的变化：登记是否更顺手，工资是否更清楚，备份和同步是否更可靠。

01 v0.2.10

手机端交互与登录修复

2026-05-13
  • 手机端云备份会读取当前设置二级页表单，登录、备份和恢复不再读不到密码。
  • 登记抽屉改为稳定的打开/关闭过渡，日期点击和设置切换不再反复整页闪现。
  • 批量处理保留当前草稿，切换添加/删除不会清空日期范围、加班小时和确认状态。
  • 管理员登录配置兼容更多 KV 写法，坏密文不再暴露底层错误。
02 v0.2.9

班次切换与设置保存

2026-05-13
  • 修复移动端设置详情页里班次管理无法切换上班方式的问题。
  • 保存设置时只重置当前表单实际包含的字段，避免其他设置被误清空。
  • 切换上班方式时局部更新班次编辑器，减少全页重绘。
03 v0.2.8

自动工时与手机二级设置

2026-05-13
  • 无记录工作日可自动按默认正班工时计入工资，减少每天重复登记。
  • 批量添加改为直接按钮处理，避免表单断开导致无反应。
  • 移动端设置页改为菜单列表和二级详情页，更接近 App 操作习惯。
04 v0.2.7

缓存穿透与版本生效修复

2026-05-13
  • 入口脚本、样式、模块导入和 Service Worker 注册都加入版本参数，避免旧 PWA 缓存继续显示上一版。
  • Service Worker 缓存升级到 v19，并跳过云同步 API 请求，减少接口被缓存干扰的风险。
  • 更新独立时间轴日志，确保侧边栏和日志页面显示同一版本。
05 v0.2.6

批量登记与移动端抽屉

2026-05-13
  • 批量添加新增补齐基础工时 8h、批量登记加班和按班次模板添加。
  • 已有加班记录时仍可补齐基础工时，底薪+加班不会重复计算底薪。
  • 移动端点击日期会弹出底部登记抽屉，日历保持完整 7 列宽度并横向滚动。
06 v0.2.5

部署、安全云备份与侧边栏收口

2026-05-13
  • 新增 `npm run build`，静态输出到 dist，方便部署到阿里云 ESA Pages。
  • 云端 RSA 私钥键名统一为 RSA_key，管理员后台独立为 admin.html。
  • 版权、动态日志、GitHub 和博客友情链接统一放到底部区域。
07 v0.2.4

底部动态日志与版权外链

2026-05-12
  • 更新日志从主导航降级为底部动态入口，减少日常记录时的干扰。
  • 底部新增 yuan 版权所有、GitHub 仓库和 yuan6.cn 博客友情链接。
  • GitHub 与博客使用从 myblog 图标资产提取的 SVG 图标，并跟随明暗主题。
08 v0.2.3

独立后台与安全云备份

2026-05-12
  • 管理员能力移到独立后台页面，日常工时应用不再出现后台入口。
  • 用户侧云备份隐藏技术配置，只保留账号、密码、备份和恢复。
  • 密码会先在浏览器内加密，再发送到云端进行校验或保存哈希。
  • 更新日志加入应用侧边栏，可以在右侧主区域直接查看。
09 v0.2.2

输入、主题与时间轴更新日志

2026-05-12
  • 金额类输入改为 step=any，不再限制只能按固定步长录入。
  • 首次引导页新增每日工时实时预览，输入上下班和午休后立即显示正班与加班。
  • 新增跟随系统、浅色、深色三种主题模式，修正深色模式下白色输入框和低对比文字。
  • 新增当前时间轴更新日志页面，整体风格参考 portfolio 的经历部分。
10 v0.2.1

云同步、后台与真实排班

2026-05-12
  • 新增自定义每周开始日、双休、单休、自定义周休、上六休一和上十四休一。
  • 新增云备份服务，支持多设备保存和恢复数据。
  • 新增用户看板，可查看用户数、IP 网段分布、同步/登录使用情况并改密或停用用户。
11 v0.2.0

数据可视化与深色适配

2026-05-12
  • 报表页新增薪资构成饼图和周工时趋势图。
  • 记录页新增搜索和日期范围筛选。
  • 日历每周底部显示工时和工资小计。

未补登工作日

2026-05-12

  • 点击日期即可跳到当天登记，移动端使用横向滚动列表。

2026-05-12
  • 新增首次使用向导，引导配置薪资方式、底薪和班次。
  • 设置页改为分组折叠，减少滚动和认知负担。
14 v0.1.6

每日登记交互优化

2026-05-12
